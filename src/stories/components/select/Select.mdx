# Select

Select components reduce cognitive load during choice-making through intelligent choice architecture and progressive disclosure. Built on the principle that decision-making should feel confident and efficient, especially when choices have consequences.

<Canvas of={Stories.Common} />

## Usage Guidelines

Select components should simplify complex choices through intelligent organization, contextual information, and progressive disclosure. The primary purpose is to guide users toward confident decisions without overwhelming them with options.

#### DOs
- **Choice Architecture**
  - Show item counts to set proper expectations
  - Enable search for lists with 8+ options
  - Group related options with clear labels
  - Provide contextual descriptions for complex choices
- **Trust Building**
  - Use progressive disclosure to reduce cognitive load
  - Include keyboard shortcuts for power users
  - Provide clear visual feedback during selection
  - Ensure selections are easily reversible
- **Decision Support**
  - Organize options in logical groupings
  - Use descriptive labels that eliminate ambiguity
  - Show additional context when choices have consequences
  - Guide users toward appropriate selections

#### DON'Ts
- **Choice Overload**
  - Don't present more than 12 ungrouped options
  - Avoid unclear or ambiguous option labels
  - Don't hide search functionality when needed
  - Avoid presenting choices without proper context
- **Cognitive Burden**
  - Never use jargon without explanation
  - Don't make users guess what options do
  - Avoid inconsistent grouping or organization
  - Don't present choices in random or confusing order
- **Accessibility Violations**
  - Don't create inaccessible custom dropdown interfaces
  - Avoid insufficient touch target sizes
  - Don't rely solely on color to convey option meaning
  - Never break keyboard navigation expectations

#### Use Cases
- **Simple Selection**: Country picker, status selection, category choice
- **Complex Decisions**: Product configuration, action selection with consequences
- **Progressive Disclosure**: Multi-level categorization, filtered option sets
- **Power User Tools**: Command palettes, quick actions with shortcuts
- **Form Integration**: Settings configuration, preference updates, data entry

#### Trust Levels
- **Low**: Simple preferences, reversible choices, UI settings
- **Medium**: Content categorization, workflow status, notification preferences  
- **High**: Data export formats, privacy settings, account configuration
- **Critical**: Destructive action selection, permanent configuration changes

## Intelligence Features

### Choice Architecture
- **Item Counting**: Shows total options to set user expectations
- **Progressive Search**: Automatically enables search for large option sets
- **Logical Grouping**: Organizes related options with clear labels
- **Contextual Descriptions**: Provides additional information for complex choices

### Motor Accessibility  
- **Enhanced Touch Targets**: Minimum 44px height for mobile accessibility
- **Size Variants**: Default and large sizes for different contexts
- **Clear Visual Feedback**: Hover, focus, and selection states
- **Consistent Interaction**: Follows native select behavior expectations

### Cognitive Load Management
- **Smart Defaults**: Intelligent placeholder text and initial selections
- **Progressive Disclosure**: Reveals complexity gradually as needed
- **Search Integration**: Built-in filtering for large option sets
- **Keyboard Shortcuts**: Power user efficiency without cluttering interface
