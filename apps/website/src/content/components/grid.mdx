## Usage Patterns

### ✅ DO
- **Linear:** Product catalogs, image galleries, equal-priority content
- **Golden:** Editorial layouts, feature showcases, natural hierarchy
- **Bento:** Editorial layouts, dashboards, content showcases (use sparingly)
- **Custom:** Specialized layouts requiring specific configurations

### ❌ NEVER
- Decorative asymmetry without semantic meaning
- Exceed cognitive load limits (8 items max on wide screens)

## Bento Patterns

The Bento preset includes semantic patterns for content-driven asymmetric layouts:

| Pattern | Use Case | Grid Template | Primary Span |
|---------|----------|---------------|--------------|
| `editorial` | Hero article + supporting content | 3 cols × 2 rows | 2×2 span |
| `dashboard` | Primary metric + supporting data | 4 cols × 2 rows | 2×1 span |
| `feature-showcase` | Main feature + benefits | 3 cols × 3 rows | 2×2 span |
| `portfolio` | Featured work + gallery items | 4 cols × 3 rows | 2×2 span |

## Gap Spacing

Control spacing using design system tokens:

| Gap | Size | Use Case |
|-----|------|----------|
| `xs` | 8px | Tight layouts, compact cards |
| `sm` | 12px | Dense information displays |
| `md` | 16px | Standard content spacing |
| `lg` | 24px | Comfortable reading layouts |
| `xl` | 32px | Generous breathing room |
| `comfortable` | 1.618rem | Golden ratio spacing |
| `generous` | 2.618rem | Golden ratio squared |

## GridItem Priority System

For Bento layouts, items can declare content priority:

| Priority | Attention Weight | Typical Span | Use Case |
|----------|-----------------|--------------|----------|
| `primary` | 8/10 | 2×2 | Hero content, main focus |
| `secondary` | 4/10 | 1×1 | Supporting content |
| `tertiary` | 2/10 | 1×1 | Contextual content |

```tsx
<Grid preset="bento" bentoPattern="editorial">
  <GridItem priority="primary">
    <article>Hero Article Content</article>
  </GridItem>
  <GridItem priority="secondary">
    <aside>Supporting Information</aside>
  </GridItem>
  <GridItem priority="tertiary">
    <nav>Related Links</nav>
  </GridItem>
</Grid>
```

## Responsive Behavior

All presets include intelligent responsive breakpoints:

```tsx
// Linear preset responsive classes
base: 'grid-cols-1'      // Mobile: single column
sm: 'sm:grid-cols-2'     // Small: 2 columns  
md: 'md:grid-cols-2'     // Medium: 2 columns
lg: 'lg:grid-cols-3'     // Large: 3 columns
xl: 'xl:grid-cols-4'     // Extra large: 4 columns
```

## Custom Configurations

For specialized layouts, use the custom preset:

```tsx
<Grid 
  preset="custom" 
  columns={{ base: 1, md: 2, lg: 3 }}
  autoFit="md"
  gap="lg"
>
  <GridItem colSpan={2}>Wide Content</GridItem>
  <GridItem>Regular Content</GridItem>
</Grid>
```
