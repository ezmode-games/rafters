# Container Component

The Container component is an invisible structural element that controls content width and breathing room without competing for attention. It provides semantic HTML structure with intelligent defaults based on element type.

## Design Intelligence

**Cognitive Load: 0/10** - Invisible structure that reduces visual complexity rather than adding to it.

**Trust Building:** Predictable boundaries and consistent spacing patterns create user confidence through systematic layout behavior.

**Semantic Meaning:** Content width control with contextual intelligence:
- `as="main"` → Primary content container
- `as="section"` → Thematic content grouping  
- `as="article"` → Standalone reading content

## Usage Patterns

### ✅ DO
- Use padding prop for internal breathing room
- Control content boundaries with semantic max-width
- Apply semantic structure with `as="main|section|article"`
- Maintain predictable component boundaries

### ❌ NEVER
- Use margins for content spacing (use padding instead)
- Unnecessarily nest containers or use fixed widths

## Semantic Intelligence

The Container component automatically applies intelligent defaults based on the semantic HTML element:

| Element | Default Size | Default Padding | Special Features | Use Case |
|---------|-------------|-----------------|------------------|-----------|
| `main` | `full` | `8` | Container queries enabled | Primary content, takes full width |
| `article` | `4xl` | `6` | **Automatic theme typography** | Optimal reading width (~65 characters) |
| `section` | `5xl` | `4` | Container queries enabled | Thematic grouping, moderate width |
| `div` | `4xl` | `4` | Basic container | Generic container |

## Automatic Article Styling

When using `as="article"`, the Container renders as an HTML `<article>` element, which automatically receives typography styling from the theme's article tag styles:

- **Typography Hierarchy:** Optimized heading sizes, line-heights, and spacing
- **Reading Flow:** Proper paragraph spacing and text flow patterns  
- **Accessibility:** Enhanced contrast and readable font sizing
- **Design System Integration:** Consistent with typography tokens

```tsx
<Container as="article">
  {/* Automatic article typography from theme */}
  <h1>Main Heading</h1>      {/* Styled by article h1 rules */}
  <p>Body paragraph...</p>    {/* Styled by article p rules */}
  <h2>Section Heading</h2>    {/* Styled by article h2 rules */}
  <blockquote>Quote</blockquote> {/* Styled by article blockquote rules */}
</Container>
```

This semantic approach enables the Container to understand content type and apply appropriate styling automatically through native HTML semantics, reducing the need for additional CSS classes.

| `5xl` | 64rem | Wide content layouts |
