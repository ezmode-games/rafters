# Badge Component

Compact status indicator that provides quick visual information without disrupting content flow.

<ComponentPreview 
  component="badge" 
  showIntelligence={true}
  height="200px"
  client:load
/>

## Usage Guidelines

| DO | NEVER |
|:---|:------|
| Use consistent badge patterns across similar content | Multiple badges competing for attention on single items |
| Keep badge text concise (1-2 words maximum) | Long text that breaks badge visual integrity |
| Apply semantic colors that match user expectations | Decorative badges without functional purpose |
| Position badges where they support scanning patterns | Critical information solely in badge format |

## Accessibility Features

- **WCAG AAA Compliance:** Sufficient color contrast ratios
- **Screen Reader Support:** Meaningful status announcements
- **Color Independence:** Clear visual patterns beyond color alone
- **Keyboard Navigation:** Focusable when interactive

## Semantic Patterns

| Status | Color | Use Case |
|--------|-------|----------|
| Active | Success | Live, published, enabled |
| Pending | Warning | Review needed, processing |
| Inactive | Default | Draft, disabled, archived |
| Error | Destructive | Failed, blocked, invalid |
