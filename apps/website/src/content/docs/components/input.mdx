# Input Component

Form input component with intelligent validation states and embedded accessibility patterns.

<r-component-preview component="input" variant="default" props='{"type":"text","placeholder":"Enter text..."}'>
</r-component-preview>

## Usage Guidelines

| DO | NEVER |
|:---|:------|
| Use clear, descriptive placeholder text | Placeholder text as the only label |
| Provide immediate validation feedback | Ambiguous error messages |
| Group related inputs with consistent spacing | Validation on every keystroke for long inputs |
| Use appropriate input types for data format | Required fields without clear indication |

## Accessibility Features

- **WCAG AAA Compliance:** Proper labeling and error association
- **Keyboard Navigation:** Tab order and focus management
- **Screen Reader Support:** ARIA labels and error announcements
- **High Contrast:** Clear visual boundaries and state indicators

## Validation Patterns

| Input Type | Validation Timing | Pattern |
|------------|------------------|---------|
| Email | On blur | Format validation |
| Password | On input | Strength indicator |
| Required fields | On submit | Presence validation |
| Optional fields | On blur | Format validation |
