# Card Component

Content container that groups related information with intelligent spacing and visual hierarchy.

  component="card" 
  showIntelligence={true}
  height="300px"
  client:load
/>

## Usage Guidelines

| DO | NEVER |
|:---|:------|
| Group related information within single cards | Nest cards within cards (increases cognitive load) |
| Use consistent card spacing in grid layouts | Mix interactive and static cards in same context |
| Maintain clear content hierarchy within cards | Overcrowd cards with unrelated content |
| Apply hover states for interactive cards | Use cards for single data points |

## Accessibility Features

- **WCAG AAA Compliance:** Proper heading structure and semantic markup
- **Keyboard Navigation:** Focus management for interactive cards
- **Screen Reader Support:** Meaningful content grouping and labeling
- **Touch Targets:** 44px minimum for interactive elements within cards

## Content Patterns

| Content Type | Recommended Structure | Cognitive Load |

| Profile | Avatar + Name + Role + Actions | 2/10 |
