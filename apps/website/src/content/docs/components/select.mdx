# Select Component

Dropdown selection component that manages option complexity through progressive disclosure.

<ComponentPreview 
  component="select" 
  showIntelligence={true}
  height="250px"
  client:load
/>

## Usage Guidelines

| DO | NEVER |
|:---|:------|
| Limit options to 7±2 items when possible (Miller's Law) | Long lists without grouping or search |
| Use clear, descriptive option labels | Ambiguous option labels |
| Group related options with meaningful categories | Multiple select for single-choice scenarios |
| Provide search for large option sets | Critical actions buried in option lists |

## Accessibility Features

- **WCAG AAA Compliance:** Keyboard navigation and screen reader support
- **Focus Management:** Proper focus trapping within dropdown
- **ARIA Support:** Role, state, and property announcements
- **Keyboard Shortcuts:** Arrow keys, Enter, Escape, typing to filter

## Cognitive Load Management

| Option Count | Strategy | Cognitive Load |
|--------------|----------|----------------|
| 2-7 items | Simple dropdown | 2/10 |
| 8-20 items | Grouped options | 4/10 |
| 21+ items | Search + grouping | 6/10 |
| Complex data | Multi-level select | 7/10 |
