# Button Component

Interactive action triggers with embedded design intelligence for trust-building and attention management.

<r-component-preview component="button" variant="default">
  Default Button
</r-component-preview>

## Usage Guidelines

| DO | NEVER |
|:---|:------|
| Use primary buttons for main user goals (maximum 1 per section) | Multiple primary buttons competing for attention |
| Group related actions with consistent spacing | Vague labels like "Click here" or "Submit" |
| Provide clear action labels that describe the outcome | Destructive actions without confirmation |
| Use destructive variant for irreversible actions | Primary buttons for low-priority actions |

## Accessibility Features

- **WCAG AAA Compliance:** 44px minimum touch targets
- **Keyboard Navigation:** Focus indicators and Enter/Space activation
- **Screen Reader Support:** Descriptive labels and ARIA attributes
- **Color Independence:** Clear visual hierarchy beyond color alone

## Trust Building Patterns

| Action Type | Trust Level | Pattern Required |
|-------------|-------------|------------------|
| Save changes | Low | Direct action |
| Delete item | Medium | Single confirmation |
| Delete account | High | Progressive confirmation |
| Data export | Low | Direct action |
