# Toast Component

Temporary notification that provides contextual feedback and system status updates without disrupting user workflow.

## Usage Guidelines

### DO

- Use for immediate feedback on user actions
- Keep messages concise and actionable
- Provide appropriate auto-dismiss timing
- Stack multiple toasts with proper spacing

### NEVER

- Use for critical information that must be read
- Create toasts that block important UI elements
- Use identical messages repeatedly
- Make dismissal mechanisms hard to find

## Accessibility Features

- **WCAG AAA Compliance:** Proper ARIA live regions for screen reader announcements
- **Keyboard Navigation:** Focusable dismiss button and escape key support
- **Screen Reader Support:** Automatic announcement of toast content
- **Timing Controls:** Pause on hover and focus for users who need more time

## Toast Timing

| Message Type | Auto-Dismiss Time | Reasoning |
|--------------|------------------|-----------|
| Success | 3-4 seconds | Quick confirmation |
| Info | 5-6 seconds | More reading time |
| Warning | 8-10 seconds | Requires attention |
| Error | Manual dismiss | Critical information |
