# OKLCH Color System

Rafters uses the OKLCH color space to create perceptually uniform color relationships with embedded semantic meaning. Unlike traditional RGB or HSL systems, OKLCH ensures that equal numeric changes produce equal visual changes.

## Why OKLCH?

### Perceptual Uniformity
OKLCH (Oklch: Lightness, Chroma, Hue) is based on human color perception:
- **L=50 always looks like 50% lightness** across all hues
- **Equal chroma values produce equal visual saturation** regardless of hue
- **Hue rotation produces natural color harmonies** without brightness shifts

### Predictable Relationships
```css
/* All these colors have identical visual brightness */
oklch(50% 0.15 0)    /* Red at 50% lightness */
oklch(50% 0.15 120)  /* Green at 50% lightness */
oklch(50% 0.15 240)  /* Blue at 50% lightness */
```

### Better Dark Modes
OKLCH maintains color relationships across light and dark themes:
```css
/* Light mode */
--primary: oklch(45% 0.25 250);

/* Dark mode - same hue and chroma, adjusted lightness */
--primary: oklch(65% 0.25 250);
```

## Color Intelligence

### Cognitive Load Through Color

Colors carry cognitive weight that affects mental processing:

**Low Cognitive Load Colors**:
- Neutral grays: `oklch(50% 0.02 250)`
- Subtle backgrounds: `oklch(96% 0.005 250)`
- Muted text: `oklch(40% 0.008 250)`

**Medium Cognitive Load Colors**:
- Branded primaries: `oklch(55% 0.20 250)`
- Status indicators: `oklch(55% 0.18 145)`
- Interactive elements: `oklch(60% 0.15 250)`

**High Cognitive Load Colors**:
- Error/warning states: `oklch(55% 0.25 25)`
- High contrast alerts: `oklch(95% 0.05 25)`
- Critical actions: `oklch(45% 0.28 25)`

### Trust Building Through Color

Color choices communicate trust levels:

**Trust-Building Blues** (250° hue):
- Professional, reliable, systematic
- Primary actions, information display
- `oklch(55% 0.22 250)` for standard trust

**Warning Ambers** (65° hue):
- Caution without alarm
- Reversible actions with consequences
- `oklch(65% 0.18 65)` for moderate warning

**Critical Reds** (25° hue):
- High consequence, irreversible actions
- Error states, destructive operations
- `oklch(55% 0.25 25)` for critical warning

## Accessibility Integration

### WCAG AAA Compliance

All color combinations meet WCAG AAA standards (7:1 contrast ratio):

```css
/* Guaranteed accessible combinations */
.primary-button {
  background: oklch(45% 0.25 250);     /* Primary blue */
  color: oklch(98% 0.02 250);          /* 7.2:1 contrast ratio */
}

.destructive-button {
  background: oklch(40% 0.28 25);      /* Critical red */
  color: oklch(98% 0.02 25);           /* 8.1:1 contrast ratio */
}
```

### Reduced Motion Considerations

Color transitions respect `prefers-reduced-motion`:

```css
.color-transition {
  transition: background-color 0.2s ease;
}

@media (prefers-reduced-motion: reduce) {
  .color-transition {
    transition: none;
  }
}
```

### Systematic Color Generation

Use the Rafters color intelligence system:

```bash
# Generate semantically meaningful color scales
npx rafters color generate --base="oklch(55% 0.22 250)" --semantic="primary"

# Create accessible color combinations
npx rafters color contrast --background="primary-600" --minimum="AAA"
```

### Dark Mode Strategy

OKLCH enables intelligent dark mode adaptation:

```css
/* Automatic dark mode with maintained relationships */
@media (prefers-color-scheme: dark) {
  :root {
    --color-primary: oklch(65% 0.22 250);      /* Lighter for dark backgrounds */
    --color-background: oklch(8% 0.004 250);   /* True dark background */
    --color-foreground: oklch(95% 0.006 250);  /* High contrast text */
  }
}
```

This systematic approach ensures that colors support user understanding rather than merely providing visual decoration. Every color choice communicates meaning and contributes to the overall user experience.
