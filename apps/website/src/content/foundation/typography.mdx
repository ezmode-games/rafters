# Typography System

Rafters uses IBM Plex Sans and IBM Plex Mono to create a distinctive technical aesthetic that signals "this is for builders" while maintaining excellent readability for all content types.

## Font Selection Strategy

### IBM Plex Sans - Body Text
**Purpose**: Optimized for extended reading and UI text
- **Weight Range**: 100-700 (Light to Bold)
- **Default Weight**: 300 (Light) for reduced visual weight
- **Line Height**: 1.6-1.7 for comfortable reading
- **Use Cases**: Paragraphs, descriptions, labels, form inputs

### IBM Plex Mono - Headings & Code
**Purpose**: Technical identity and code display
- **Weight Range**: 100-700 (Light to Bold) 
- **Default Weight**: 400 (Regular) for proper mono spacing
- **Line Height**: 1.2 for tight heading hierarchy
- **Use Cases**: Headings, code blocks, technical labels

## Hierarchical Scale

### Heading Hierarchy
```css
/* H1 - Page Titles */
h1 { 
  font-family: var(--font-mono);
  font-size: 2.25rem;      /* 36px */
  line-height: 1.2;
  font-weight: 400;
  margin-top: 0;
}

/* H2 - Major Sections */
h2 { 
  font-family: var(--font-mono);
  font-size: 1.875rem;     /* 30px */
  line-height: 1.2;
  font-weight: 400;
  border-bottom: 1px solid #e5e7eb;
  padding-bottom: 0.5rem;
}

/* H3 - Sub-sections */
h3 { 
  font-family: var(--font-mono);
  font-size: 1.5rem;       /* 24px */
  line-height: 1.2;
  font-weight: 400;
}

/* H4-H6 - Minor headings */
h4, h5, h6 { 
  font-family: var(--font-mono);
  font-size: 1.25rem;      /* 20px */
  line-height: 1.2;
  font-weight: 400;
}
```

## Mathematical Relationships

### Modular Scale
Typography follows a **1.2 ratio (Minor Third)** for harmonious proportional relationships:

- **12px** (0.75rem) - Captions
- **14px** (0.875rem) - Secondary text  
- **16px** (1rem) - Base size
- **18px** (1.125rem) - Body text
- **20px** (1.25rem) - Large body
- **24px** (1.5rem) - H3
- **30px** (1.875rem) - H2
- **36px** (2.25rem) - H1

### Line Height Strategy
- **Headings**: 1.2 (tight for visual hierarchy)
- **Body Text**: 1.6-1.7 (comfortable reading)
- **UI Elements**: 1.25-1.5 (compact interfaces)
- **Code**: 1.4 (readable but compact)

## Accessibility & Performance

### WCAG Compliance
- **Minimum font sizes**: 16px base (never below 14px)
- **Contrast ratios**: Minimum 4.5:1 (AA), target 7:1 (AAA)
- **Line length**: 45-75 characters for optimal reading
- **Line height**: Minimum 1.5 for body text

## Design Intelligence Integration

### Cognitive Load Through Typography
**Low Cognitive Load** (0-2/10):
- Light font weights (300)
- Generous line spacing (1.6-1.7)
- Comfortable font sizes (16px+)
- High contrast ratios

**Medium Cognitive Load** (3-5/10):
- Regular font weights (400-500)
- Standard line spacing (1.4-1.6)
- Smaller font sizes (14-16px)
- Clear hierarchy

**High Cognitive Load** (6+/10):
- Bold font weights (600-700)
- Tight line spacing (1.2-1.4)
- Dense information display
- Multiple font sizes in proximity

### Trust Building Through Typography
- **Professional**: IBM Plex Mono for headings signals technical expertise
- **Readable**: IBM Plex Sans ensures accessibility and comfort
- **Systematic**: Mathematical scale demonstrates design precision
- **Consistent**: Same fonts across all brand touchpoints

This typography system supports both human readers and AI agents by providing clear semantic meaning, systematic relationships, and embedded reasoning about cognitive load and trust building.
