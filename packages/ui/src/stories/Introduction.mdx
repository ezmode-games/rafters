# Rafters Design System

Rafters is a trust-building component system designed for consequential interfaces. Built on the principle that design decisions should reduce cognitive load and increase user confidence, especially during critical moments.

## Philosophy

**Trust-building through intelligence.** Every component in Rafters implements three layers of intelligence:

1. **Visual Intelligence** - Progressive visual hierarchy based on consequence and trust levels
2. **Behavioral Intelligence** - Smart defaults and adaptive patterns that guide users naturally 
3. **Accessibility Intelligence** - Universal design that works for everyone, not just compliance

## Trust-Building Principles

### Progressive Trust Levels

Every component implements four trust levels that match user psychology:

- **Low Trust** - Routine actions, minimal friction, reversible
- **Medium Trust** - Moderate consequences, balanced caution
- **High Trust** - Significant impact, deliberate friction
- **Critical Trust** - Permanent consequences, maximum friction

### Attention Economics

Visual hierarchy follows consequence severity:
- Primary actions get strongest visual weight
- Secondary actions provide clear alternatives  
- Escape routes are always clearly available
- Destructive actions require appropriate friction

### Cognitive Load Management

Information architecture reduces mental overhead:
- Progressive disclosure for complex interactions
- Clear information hierarchy
- Consistent interaction patterns
- Contextual help and guidance

## Implementation Examples

### Button Component
**Trust-building through visual hierarchy**
```tsx
// Low-stakes action
<Button variant="outline" size="sm">Save Draft</Button>

// High-stakes action  
<Button variant="destructive" destructiveConfirm>Delete Account</Button>
```

### Dialog Component
**Progressive confirmation for critical actions**
```tsx
// Critical trust level with enhanced confirmation
<Dialog trustLevel="critical" destructive>
  <DialogContent>
    <DialogTitle>Delete Account Forever</DialogTitle>
    <DialogDescription>
      This action cannot be undone. This will permanently delete 
      your account and all data.
    </DialogDescription>
  </DialogContent>
</Dialog>
```

## Development Workflow

### Component Creation Process

1. **Define Trust Context** - What are the consequences of this interaction?
2. **Map User Journey** - Where does anxiety exist? How can we reduce it?
3. **Implement 7-File Pattern** - Complete documentation and story coverage
4. **Test Accessibility** - Ensure universal usability
5. **Validate Trust Patterns** - Does it feel safe and clear?
