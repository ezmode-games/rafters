# Container

Foundational layout container that provides intelligent content width, spacing, and semantic structure. Container components use design system tokens and semantic HTML elements with automatic defaults optimized for accessibility and readability.

<Canvas of={ContainerStories.Default} />

## Usage Guidelines

Containers establish content boundaries and provide consistent spacing using design system tokens. They support semantic HTML elements that automatically apply appropriate defaults for different content types.

#### DOs
- **Semantic HTML**
  - Use `as="main"` for primary content areas and application wrappers
  - Use `as="article"` for standalone content like blog posts and documentation  
  - Use `as="section"` for thematic content groupings and feature sections
  - Use `as="div"` only when semantic meaning isn't clear
- **Spacing Philosophy**
  - Use padding prop for internal spacing and content breathing room
  - Use `mx-auto` for centering containers (only margin exception allowed)
  - Rely on semantic defaults - only override when necessary
- **Container Queries**
  - Prefer container queries (`@sm:`, `@md:`, `@lg:`) over viewport queries
  - Enable container queries for component-responsive behavior
  - Use single container query contexts to serve multiple children

#### DON'Ts  
- **Spacing Anti-patterns**
  - Don't use margins for content spacing (use padding instead)
  - Don't nest containers unnecessarily when a single container suffices
  - Don't mix arbitrary spacing values with design system tokens
- **Semantic Misuse**
  - Don't use generic `div` when semantic elements like `main`, `article`, or `section` apply
  - Don't override container widths with custom CSS
  - Don't use `size="full"` unless you need true edge-to-edge layouts

#### Use Cases
- **Main Containers**: Application layouts, page wrappers, primary content areas
- **Reading Content**: Articles, blog posts, documentation, long-form text
- **Content Sections**: Feature areas, thematic groupings, component sections  
- **Flexible Layouts**: Custom containers where semantic meaning isn't applicable
