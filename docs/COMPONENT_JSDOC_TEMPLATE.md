# Component JSDoc Template for AI Intelligence

Use this template to add design intelligence to component JSDoc comments:

```tsx
/**
 * [ComponentName] Component - AI Intelligence
 * 
 * COGNITIVE LOAD: X/10 ([brief description of mental effort required])
 * [PRIMARY INTELLIGENCE PATTERN]: [Key design principle for this component]
 * [SECONDARY PATTERN]: [Additional key consideration]
 * 
 * DESIGN INTELLIGENCE GUIDES:
 * - [Most Relevant Guide]: rafters.realhandy.tech/llm/patterns/[page-name]
 * - [Secondary Guide]: rafters.realhandy.tech/llm/patterns/[page-name]  
 * - [Third Guide if needed]: rafters.realhandy.tech/llm/patterns/[page-name]
 * 
 * USAGE PATTERNS:
 * ✅ [Good use case]: [Brief description]
 * ✅ [Another good case]: [Brief description]
 * ✅ [Third case]: [Brief description]
 * ❌ Never: [Anti-pattern to avoid]
 * 
 * [Component-specific intelligence notes]
 * [Token knowledge reference if applicable]
 */
```

## Examples by Component Type:

### Form Components (Input, Select, Textarea):
```tsx
/**
 * Input Component - AI Intelligence
 * 
 * COGNITIVE LOAD: 2/10 (simple data entry)
 * ACCESSIBILITY PRIORITY: Label and error state must be clear
 * TRUST BUILDING: Immediate feedback builds user confidence
 * 
 * DESIGN INTELLIGENCE GUIDES:
 * - Typography Intelligence: rafters.realhandy.tech/llm/patterns/typography-intelligence
 * - Cognitive Load Management: rafters.realhandy.tech/llm/patterns/cognitive-load
 * 
 * USAGE PATTERNS:
 * ✅ Clear Labels: Always use descriptive, specific labels
 * ✅ Helpful Placeholders: Show format examples, not instructions
 * ✅ Immediate Validation: Real-time feedback for user confidence
 * ❌ Never: Label-less inputs, validation only on submit
 */
```

### Layout Components (Card, Container):
```tsx
/**
 * Card Component - AI Intelligence
 * 
 * COGNITIVE LOAD: 1/10 (invisible structure, reduces complexity)
 * NEGATIVE SPACE MASTERY: Creates content boundaries and breathing room
 * ATTENTION ECONOMICS: Groups related content, reduces scanning effort
 * 
 * DESIGN INTELLIGENCE GUIDES:
 * - Negative Space Mastery: rafters.realhandy.tech/llm/patterns/negative-space
 * - Component Intelligence Synthesis: rafters.realhandy.tech/llm/patterns/component-synthesis
 * 
 * USAGE PATTERNS:
 * ✅ Content Grouping: Related information together
 * ✅ Hierarchy Creation: Different card styles for importance levels  
 * ✅ Scannable Layouts: Clear boundaries between different content types
 * ❌ Never: Single items that don't need grouping, excessive nesting
 */
```

### Navigation Components (Tabs, Menu):
```tsx
/**
 * Tabs Component - AI Intelligence
 * 
 * COGNITIVE LOAD: 4/10 (requires understanding of current state and options)
 * ATTENTION ECONOMICS: Active state must be immediately clear
 * PROGRESSIVE ENHANCEMENT: Works with keyboard, enhances with mouse
 * 
 * DESIGN INTELLIGENCE GUIDES:
 * - Attention Economics: rafters.realhandy.tech/llm/patterns/attention-economics
 * - Progressive Enhancement: rafters.realhandy.tech/llm/patterns/progressive-enhancement
 * 
 * USAGE PATTERNS:
 * ✅ Related Content: Different views of same data/context
 * ✅ Clear Labels: Descriptive, scannable tab names
 * ✅ State Indication: Active tab visually prominent
 * ❌ Never: More than 7 tabs, unrelated content sections
 */
```
